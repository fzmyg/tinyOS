# TinyOS —— 一个用于系统学习的微型操作系统内核

TinyOS 是一个基于 x86 架构实现的教学型操作系统内核，完整覆盖了从 **Boot 启动 → 内核初始化 → 内存管理 → 进程调度 → 文件系统 → 用户态程序运行** 的核心流程。

该项目旨在深入理解操作系统底层机制，而非依赖现有 OS 框架。

## 项目特性

TinyOS 实现了以下核心功能：

-  Boot 引导加载（MBR + Loader + Kernel）
-  分页机制与虚拟内存管理（两级页表 + 页表自映射）
-  线程调度（时间片轮转 + 抢占式调度）
-  用户进程支持（独立页表 + 特权级切换）
-  ELF 可执行文件加载
-  fork / exec / wait / exit 进程模型
-  信号量 + 可重入锁同步机制
-  键盘 / IDE 磁盘 / 控制台驱动
-  简易 inode 文件系统
-  Shell 命令解析程序
-  用户态仿 vim 文件编辑器

## 项目目录结构

- ./bin 中存放bochs虚拟机程序
- ./share以及./bochsrc 中存放bochs虚拟机配置文件以及虚拟磁盘文件
- ./notes 中存放学习过程笔记
- ./src 中存放OS内核**源码**
- ./hd60M.img 为OS启动盘（存放MBR,Loader以及OS内核）
- ./hd80M.img 为OS运行时存储盘
- ./Makefile 用与组织内核编译
- ./boot.sh 用与启动编译并初始化OS

## 运行教程

```shell
cd 项目根目录
chmod +x boot.sh
./boot.sh
make boot
```

bochs若为gdb版本还需执行如下指令：

```shell
gdb
target remote localhost:1234
continue
```

## 源码分析

### boot ---  引导程序

启动流程：BIOS → MBR → Loader → Kernel

1. 从启动介质（BIOS/固件）开始初始化硬件并加载MBR
2. MBR 中负责加载OS loader
3. loader负责加载OS内核至内存

### device  --- 设备驱动

- console：基于互斥机制的字符输出
- ide：基于端口 IO 的磁盘驱动
- keyboard：基于生产者-消费者模型的键盘缓冲
- timer：配置 PIT 产生时钟中断，实现时间片调度
- TSS：支持特权级切换与内核栈切换

### kernel  --- 中断与系统调用

- interrupt 模块负责初始化8259A中断控制芯片以及中断描述符表，中断门的设置
- syscall 模块负责设置初始系统调用表以及提供系统调用注册函数
- init_proc.c 负责运行1号进程（shell）
- main.c 与 init.c 负责内核其余模块的初始化与运行

### fs --- 文件系统

- 采用inode管理文件
- 采用位图管理空闲磁盘块
- 提供open，write，remove等系统调用接口控制文件

### mm --- 内存管理

- 基于2级页表实现虚拟内存 + 页表自映射。
- 其中**大于 1024B** 的申请按页（page）分配：先在虚拟池分配连续虚拟页，再在物理池中分配对应物理页并建立页表映射。
- **小于等于 1024B** 的申请使用 **内存块描述符（mem_block_desc） + arena 分割**：每个描述符对应一种块大小（16、32、64...1024），描述符维护一个空闲块链表（free_list）。当某个描述符 free_list 空时，会分配一页（arena），把这一页按块大小切分为若干 mem_block 并逐一加入该描述符的 free_list，然后从 free_list 取出一个返回（“从队列取小块”）。 

### lib --- 库函数

- 实现位图，系统调用，链表，数据类型，字符串，断言，边长数组，格式化打印等库函数

### thread --- 进程管理

- 定义了 `task_struct` 作为线程的 PCB，每个线程占用一个独立的 4KB 内核页，页底存放 PCB，页顶向下增长作为内核栈，通过 `esp & 0xfffff000` 可以快速定位当前线程 PCB。线程创建时通过构造初始内核栈，伪造返回现场，使线程第一次被调度时自动执行指定函数，实现了轻量级线程启动机制。
- 调度器 `schedule()` 实现了基于优先级时间片的轮转调度：当线程时间片耗尽或主动让出 CPU 时，会将当前线程重新加入就绪队列，从就绪队列中选择下一个线程执行，并通过 `switch_to` 完成上下文切换。为保证调度过程的原子性，关键路径均在关中断状态下执行。同时系统实现了 idle 线程，当没有可运行线程时通过 `hlt` 指令进入低功耗等待状态。
- 同步机制方面，代码实现了基于信号量的阻塞唤醒模型，并在此基础上封装了支持可重入的锁（通过 holder 和 holder_repeat_nr 解决同线程重复加锁问题）。信号量内部维护等待队列，当资源不可用时线程阻塞并加入等待队列，释放时唤醒阻塞线程，体现了典型的 P/V 操作模型。
- 此外还实现了一个基于环形缓冲区的 IO 队列，用于解决生产者-消费者问题。队列通过读写指针管理缓冲区状态，在队列空或满时分别阻塞消费者或生产者线程，并通过锁与线程阻塞机制保证并发安全。

### proc --- 进程装入与回收

- exec实现了ELF文件装入内存并生成相关进程
- fork 实现了进程的复制
- process.c/.h 实现了用户进程的独立页表、用户虚拟地址空间管理，以及通过中断返回机制完成从内核态到用户态的切换
- wait/exit 实现了进程的回收

### shell --- 命令解析程序

- 实现命令解析功能

### usrp --- crt库 以及仿vim文件编辑程序

- crt库用于链接程序为可执行文件
- 仿vim用于编辑文件
